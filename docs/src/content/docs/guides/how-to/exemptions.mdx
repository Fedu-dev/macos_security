---
title: Exemptions
description: An exemptions reference.
---

Use the `org.{baseline}.audit.plist` to set exemptions for rules that your organization has approved to be configured out of compliance. Exemptions allow these systems to report as compliant even when the setting differs from the rule's expected result. The check still appears as a finding in the logs and `.plist` file, but you can mark it as exempt so reporting tools exclude it from non-compliance reports.

You can place this file at `/Library/Preferences/org.{baseline}.audit.plist` or apply it through a custom configuration profile.

To set an exemption for a rule, set `exempt` to true and add an `exempt_reason`. To set an exemption for the `auth_smartcard_enforce` rule, run the following:

```
sudo /usr/libexec/PlistBuddy -c "Add :auth_smartcard_enforce:exempt bool true" /Library/Preferences/org.800-53r5_moderate.audit.plist
sudo /usr/libexec/PlistBuddy -c "Add :auth_smartcard_enforce:exempt_reason string 'Reader Issues'" /Library/Preferences/org.800-53r5_moderate.audit.plist

/usr/libexec/PlistBuddy -c "Print auth_smartcard_enforce" /Library/Preferences/org.800-53r5_moderate.audit.plist
Dict {
    finding = true
    exempt = true
    exempt_reason = Reader Issues
}
```

**📌 NOTE**\
If `org.{baseline}.audit.plist` is not set, the script defaults `exempt` to false.
