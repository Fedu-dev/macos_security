The project makes it simple for organizations to construct a tailored benchmark. You can start from one of the NIST 800-53 control baselines or customize an already established benchmark. Tailoring a benchmark is different than customizing specific rules. See [Customization](https://github.com/usnistgov/macos_security/wiki/Customization) to learn more.

**Organization Defined Values**

Organization Defined Values, or ODVs, are values determined for controls that meet both the security and functional requirements in an organization. Benchmark authors like DISA and CIS provide these values for their respective published guidance. The project includes recommended values, as well as the defined values for the DISA STIG and CIS Benchmarks, as part of the ODV field in the YAML.

## Tailoring a Benchmark

Running the generate_baseline script with the `-t` argument steps you through the tailoring process. The process starts by prompting for the following:

* Benchmark Name
* Author’s Name
* Organization

```
➜  macos_security git:(sequoia) ./scripts/generate_baseline.py -k 800-53r5_moderate -t
Enter a name for your tailored benchmark or press Enter for the default value (800-53r5_moderate): MyOrgs_Benchmark
Enter your name: Allen Golbig
Enter your organization: MyOrg
The inclusion of any given rule is a risk-based-decision (RBD).  While each rule is mapped to an 800-53 control, deploying it in your organization should be part of the decision-making process.
The script prompts you to include each rule. For rules with specific organizational defined values (ODV), it also prompts you to set those values.
```

Once completed, the script prompts you to include or exclude each rule in your benchmark. 

```
Would you like to include the rule for "audit_acls_files_configure" in your benchmark? [Y/n/all/?]:
```

**📌 NOTE**\
The `?` option displays additional details about the rule, to help you decide if it meets your organization's requirements.

If a rule contains an ODV, the script prompts you to assign a value that meets your organization’s requirements.   

```
Number of failed attempts.
Enter the ODV for "pwpolicy_account_lockout_enforce" or press Enter for the recommended value (3):
```

After the tailoring process completes, the script creates a new YAML file in `build/baselines` and custom rules containing their ODV values in `custom/rules/`. You can then run the generate guidance script against that tailored benchmark.

```
./scripts/generate_guidance.py build/baselines/MyOrgs_Benchmark.yaml -p -s -x
```
